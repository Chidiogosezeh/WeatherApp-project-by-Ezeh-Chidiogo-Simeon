DESCRIPTION: I built a weather web application using HTML, CSS, and vanilla JavaScript, structured around three separate files to keep the code organised and maintainable. The app uses the Open-Meteo API — a free, no-key-required service — to fetch real-time current conditions and a 5-day daily forecast based on geographic coordinates. For city searches, I used Open-Meteo's forward-geocoding endpoint to convert a typed city name into latitude and longitude, and BigDataCloud's reverse-geocoding API to convert browser-detected coordinates back into a readable city name. On page load, the app automatically detects the user's location using the browser's built-in navigator.geolocation API and loads local weather without any input required, falling back to a default city if permission is denied. I implemented a °C/°F unit toggle that converts temperatures client-side from the stored API response, avoiding unnecessary re-fetching. A search history feature stores the last five searched cities in localStorage, rendering them as quick-access buttons below the search bar with an option to clear them on demand. For the weather icon in the hero section, I used the Weather Icons font library combined with custom CSS keyframe animations — including a spinning sun, bouncing raindrops, and a flickering thunderstorm flash — to visually reflect current conditions. Throughout the project I focused on graceful error handling, ensuring that failed API calls, denied permissions, and invalid city names all produce clear user-facing feedback rather than silent failures.
